# How-to topic

--8<-- "docs/topic-types/single-source.md"

The purpose of a how-to topic is to walk through a procedure or workflow with the goal of completing a task, rather than learning more broadly. For example, installation instructions are how-to topics. Most procedural topics in our content will instead be tutorial topics. For additional information on how to structure how-tos, see [Diataxis](https://diataxis.fr/how-to-guides/).

## Titles

Use imperative verbs for how-to topics. For example, "Get started," "Calibrate the model," or "Configure a vaccination campaign." Avoid gerunds (-ing verbs) because they can be ambiguous, sometimes functioning as a verb, adjective, or noun. This can create confusion, especially in translation.

## Lists

Use numbered lists for the steps in a procedure for ease of scanning. If you need to include 
multiple paragraphs, sub-steps, or example code in a step, use 4-space indentation. For example,

1.  Define data stratification.

    Each observation sheet starts blank and requires you to specify observational data
    statifiers and the type of data uncertainty that will be specified.

    * Year
    * Province
    * Gender

2. Generate the data table.
